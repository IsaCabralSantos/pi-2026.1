
# Requisitos Não Funcionais — SIDERA PREDICT

---

**RNF01 — Plataforma Flutter (Mobile/Tablet)**
O sistema deve ser desenvolvido em Flutter, garantindo compatibilidade com dispositivos móveis e tablets (Android/iOS).
Critério: Interface responsiva e adaptada a diferentes tamanhos de tela.

**RNF02 — Execução Local (Edge AI)**
O processamento de dados e medições deve ocorrer localmente no dispositivo, sem dependência de nuvem.
Critério: Garantia de privacidade, segurança e operação offline.

**RNF03 — Baixa Latência (Tempo Real)**
O sistema deve fornecer resposta em tempo real, com latência máxima de 2 segundos entre captura e resultado.
Critério: Não impactar o ritmo produtivo.

**RNF04 — Interface Simples e Intuitiva**
A interface deve ser simples, com botões grandes, poucos cliques e adequada para uso com luvas e por usuários de diferentes faixas etárias.
Critério: Testes de usabilidade com operadores reais.

**RNF05 — Integração com Rede Interna**
O sistema deve integrar-se à rede interna da empresa para acesso a arquivos CAD e registro de dados.
Critério: Compatibilidade com protocolos industriais de rede.

**RNF06 — Robustez para Ambiente Industrial**
O sistema deve ser compatível com o ambiente industrial, suportando poeira, vibração e variações de temperatura.
Critério: Testes em campo e validação com usuários finais.

**RNF07 — Precisão Dimensional Confiável**
O sistema deve garantir precisão dimensional compatível com as tolerâncias técnicas exigidas pelo processo.
Critério: Validação com padrões de referência e calibração periódica.
